## 폴더 구조 (권장 컨벤션)

1인/소규모 사업자가 고객 업무를 다룰 때 권장하는 구조. `_` 접두사 = 공통/맥락, 고객은 폴더로 분리한다.

```
프로젝트/
├── BUSINESS.md         ← 사업 SSOT (서비스·가격·규칙). templates/business-ssot-template.md에서 시작
├── PERSONA.md          ← /init-persona가 생성. 판단·반응 패턴 (CLAUDE.md가 자동 로드)
├── _공통/              ← 고객 무관 공통 자료 (방법론, 템플릿)
├── clients/            ← 고객사별 폴더 (또는 `고객사/`)
│   └── {고객명}/
│       ├── _context/   ← 이 고객의 맥락·규약·소스맵 (= 고객별 SSOT)
│       ├── 기획안/      ← 산출물
│       ├── 데이터/
│       └── 사전미팅/    ← 미팅·통화 기록
└── .claude/            ← 스킬·커맨드·규칙
```

## 작업 전 필수

- 고객 작업 전 해당 `clients/{고객명}/_context/`를 먼저 읽는다.
- 사업 판단·견적·상담은 `BUSINESS.md`를 정본으로 참조한다.

## Persona (항상 적용)

`/init-persona`를 실행하면 아래 줄이 자동으로 추가되어, 매 세션 `PERSONA.md`가 로드된다.
고객 상담·판단을 흉내 낼 때는 PERSONA.md의 PART B(비즈니스 페르소나)를 체화한다.
